# Omnigram Proposals

## 背景

自己是一个技术控，自己有很多的资料（PDF、Epub、Word等），各种音视频，存储在NAS里面。时间一长文件和目录很混乱，自己如果要去搜索也很费劲，没法进行快速全局的搜索（类似windows 上everything）。我自己每天工作中查询的一些资料，通过浏览器浏览的内容也没有方法自动整理到本地存储和总结。阅读过很多很多书籍以后也没有在一个地方统一树立。只能随时间的流逝在脑海里也变得模糊。但是目前市面上并没有一款软件能进行统一的整理，于是我开始构建一个这样的工具，试图解决上述的问题（当然是出于自己的需求）。

## 早期目标和初衷（2024）

Omnigram 定位是一个实现一个多终端的文档（PDF、EPUB、音视频等）管理和阅读工具（助手）。具体而言，他支持如下的一些功能（特性）：

- 支持多终端客户端 UI（iOS、Mac、Windows、Linux、Android）；
- 支持通过 AI 模型对文字内容进行TTS，从而实现听书功能；
- 支持浏览器插件进行每日阅读总结并输出成文档存储为知识点（通过AI模型）；
- 支持多种类型资料自动管理功能（NAS服务端）和全局内容搜索；
- 对于多媒体资源如音频、视频资源提供基本的归纳和搜刮功能（类似 Jellyfin 视频管理功能，这里尽量是兼容和对接市面上成熟的音视频播放如 Infuse、Kodi 等软件）
- 支持主流网盘进行加密备份功能，定期同步本地文件到如 Google Drive、OneDrive、阿里云盘等；
- 支持基本的用户管理功能，实现家庭一些权限划分（考虑到影视分级等）；

## 战略调整（2026.03）

### 市场调研结论

经过对当前市场竞品的全面调研，发现以下格局：

#### 竞品矩阵

```
                    有服务端(NAS/自部署)          无服务端(纯客户端)
                 ┌──────────────────────┬──────────────────────┐
   有AI能力      │                      │  Anx Reader ✅       │
                 │    ❌ 空白！          │  Readest 🛠️         │
                 │                      │  Readwise (SaaS)     │
                 ├──────────────────────┼──────────────────────┤
   无AI能力      │  Calibre-Web         │  KOReader            │
                 │  Kavita              │  Foliate             │
                 │  Audiobookshelf      │                      │
                 └──────────────────────┴──────────────────────┘
```

#### 关键竞品分析

| 产品 | 定位 | 优势 | 缺失 |
|------|------|------|------|
| **Anx Reader** | Flutter AI 阅读器 | AI 助手、思维导图、TTS、多端 | 无服务端，仅 WebDAV 同步 |
| **Readest** | Tauri 跨平台阅读器 | 现代 UI、TTS、多格式 | AI 摘要开发中，无服务端 |
| **Calibre-Web** | 自部署书库 Web 管理 | 成熟社区、OPDS、用户管理 | 无 AI、无 TTS、UI 老旧 |
| **Kavita** | 自部署数字图书馆 | 漫画/书管理、内置 Web 阅读 | 无 AI、无 TTS |
| **Audiobookshelf** | 自部署有声书服务 | 有声书播放、播客、多端 App | 仅播放已有音频，不做 TTS 生成 |
| **Readwise Reader** | 阅读 + 知识管理 SaaS | AI（Ghostreader）、高亮、每日回顾 | 不可自部署，$8.99/月 |
| **Karakeep (Hoarder)** | 自部署书签 + AI | AI 标签、全文搜索 | 不做阅读器 |
| **KOReader** | 电子墨水屏阅读器 | 极致阅读体验、插件生态 | 无 AI、无服务端 |

#### 核心发现

> **「自部署/NAS + AI + 阅读管理」这个象限 = 零竞争。**
> 这正是 Omnigram 的机会窗口。

### 新定位

**Omnigram = AI-Native 自部署书库管理 + 阅读服务**

一句话描述：**给 NAS 用户做一个带 AI 的书库管理 + 阅读 + 听书服务。**

对标叙事：**Calibre-Web + AI + TTS = Omnigram**

### 核心功能（按优先级排序）

#### P0 — 核心差异化（必须做好）

1. **NAS 书库扫描与管理** — 替代 Calibre-Web 的书库管理能力
   - 多格式支持（EPUB、PDF）
   - 书籍元数据自动补全
   - OPDS 协议支持
   - 用户/家庭权限管理

2. **AI 增强阅读** — 这是无人占据的差异化
   - 书籍自动分类与智能标签
   - 全文语义搜索（embedding）— 「我记得有本书讲过 XXX」
   - 阅读时 AI 问答 / 章节摘要 / 翻译
   - 支持接入主流 LLM（OpenAI、Ollama 本地模型等）

3. **TTS 听书（服务端生成）** — Audiobookshelf 不做 TTS 生成，这是差异化
   - 从电子书文本生成有声书
   - 多 TTS 引擎后端（OpenAI TTS / Edge TTS / Fish Audio / 本地模型）
   - 分章节生成、多音色可选

4. **多端阅读器** — Flutter 全平台客户端
   - iOS / Android App
   - Web 端阅读器（重要！降低使用门槛）

#### P1 — 增强功能（第二阶段）

5. **阅读笔记与高亮** — 高亮、标注、笔记导出
6. **阅读统计** — 阅读时长、进度、习惯热力图
7. **Web 端管理后台** — 书库管理、用户管理、系统设置

#### P2 — 远期功能（降低优先级）

8. 浏览器插件（知识收集）
9. 网盘备份同步
10. 音视频管理（建议对接 Jellyfin 而非自建）

### 目标用户

- 家里有 NAS 的技术爱好者 / Homelab 玩家
- 数字书虫（大量电子书需要管理）
- 家庭用户（多账户、权限分级）
- r/selfhosted、r/homelab 社区用户

### 技术优势（已有基础）

- ✅ Go 后端 + Gin + GORM — 服务端已成型
- ✅ Flutter 全平台客户端 — 多端 App 已有框架
- ✅ TTS 集成（Fish Audio）— 听书功能已有原型
- ✅ OPDS 支持 — 标准协议对接
- ✅ OpenAPI 规范 — 前后端契约驱动开发
- ✅ Docker 部署 — 自部署友好
- ✅ 用户管理 — 多用户权限已有
