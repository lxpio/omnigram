# 001 - 竞品调研与战略定位

> 日期：2026-03-20
> 状态：✅ 已决策
> 结论：走方向 A —— 「AI-Native 自部署书库管理 + 阅读服务」

## 背景

Omnigram 项目始于 2024 年，最初定位为大而全的多终端文档管理和阅读工具。2026 年 AI 爆发后，需要重新审视市场和定位。

## 竞品矩阵

```
                    有服务端(NAS/自部署)          无服务端(纯客户端)
                 ┌──────────────────────┬──────────────────────┐
   有AI能力      │                      │  Anx Reader ✅       │
                 │    ❌ 空白！          │  Readest 🛠️         │
                 │                      │  Readwise (SaaS)     │
                 ├──────────────────────┼──────────────────────┤
   无AI能力      │  Calibre-Web         │  KOReader            │
                 │  Kavita              │  Foliate             │
                 │  Audiobookshelf      │                      │
                 └──────────────────────┴──────────────────────┘
```

## 关键竞品分析

| 产品 | 定位 | 优势 | 缺失 |
|------|------|------|------|
| **Anx Reader** | Flutter AI 阅读器 | AI 助手、思维导图、TTS、多端 | 无服务端，仅 WebDAV 同步 |
| **Readest** | Tauri 跨平台阅读器 | 现代 UI、TTS、多格式 | AI 摘要开发中，无服务端 |
| **Calibre-Web** | 自部署书库 Web 管理 | 成熟社区、OPDS、用户管理 | 无 AI、无 TTS、UI 老旧 |
| **Kavita** | 自部署数字图书馆 | 漫画/书管理、内置 Web 阅读 | 无 AI、无 TTS |
| **Audiobookshelf** | 自部署有声书服务 | 有声书播放、播客、多端 App | 仅播放已有音频，不做 TTS 生成 |
| **Readwise Reader** | 阅读 + 知识管理 SaaS | AI（Ghostreader）、高亮、每日回顾 | 不可自部署，$8.99/月 |
| **Karakeep (Hoarder)** | 自部署书签 + AI | AI 标签、全文搜索 | 不做阅读器 |
| **KOReader** | 电子墨水屏阅读器 | 极致阅读体验、插件生态 | 无 AI、无服务端 |

## 核心发现

> **「自部署/NAS + AI + 阅读管理」这个象限 = 零竞争。**

## 决策：方向 A

**Omnigram = AI-Native 自部署书库管理 + 阅读服务**

对标叙事：**Calibre-Web + AI + TTS = Omnigram**

### 功能优先级

**P0 — 核心差异化：**
1. NAS 书库扫描与管理（替代 Calibre-Web）
2. AI 增强阅读（自动分类、语义搜索、问答摘要）
3. TTS 听书（服务端生成，多引擎后端）
4. 多端阅读器（Flutter App + Web）

**P1 — 增强功能：**
5. 阅读笔记与高亮
6. 阅读统计
7. Web 管理后台

**P2 — 远期（降低优先级）：**
8. 浏览器插件
9. 网盘备份同步
10. 音视频管理（对接 Jellyfin）

### 目标用户

- NAS / Homelab 用户
- 数字书虫
- 家庭多用户场景
- r/selfhosted 社区
