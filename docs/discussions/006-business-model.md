# 006 - 商业模式分析

> 讨论日期：2026-03-20
> 状态：🔄 讨论中
> 核心问题：开源 + 自托管项目如何实现可持续商业化？

## 同类项目商业模式参考

| 项目 | 定位 | 商业模式 | 年收入估计 |
|------|------|---------|-----------|
| **Immich** (照片) | 自托管 Google Photos | 开源免费 + GitHub Sponsors | ~$30万/年（捐赠） |
| **Jellyfin** (视频) | 自托管 Plex | 纯开源，无商业化 | $0（纯社区） |
| **Plex** (视频) | 媒体服务器 | Plex Pass 订阅 $5/月 | 数千万美元 |
| **Audiobookshelf** | 有声书管理 | 开源免费 + 捐赠 | 极少 |
| **Nextcloud** | 文件协作 | 开源 + 企业版 | ~€1000万+/年 |
| **GitLab** | 代码托管 | 开源 + 企业版 SaaS | 上市公司 |
| **Outline** | 知识库 | 开源 + 托管版 $10/人/月 | 数百万美元 |
| **Logseq** | 笔记 | 开源 + Sync 订阅 | 有 VC 融资 |

### 关键教训

- ❌ 纯开源 + 捐赠 = 活不下去（Jellyfin 模式）
- ❌ 一开始就收费 = 没人用（没有社区就没有价值）
- ✅ 开源核心 + 增值服务 = 可持续（Nextcloud / Outline / Plex 模式）

## 四种可选商业模式

### 模式一：Open Core（开源核心 + 商业增值）⭐ 推荐

```
免费层（开源 AGPL）                付费层（Omnigram Pro）
┌────────────────────────┐      ┌────────────────────────┐
│ 书库管理 + 扫描导入      │      │ AI 摘要/问答（需 LLM）   │
│ OPDS/WebDAV 协议        │      │ 高质量 TTS 有声书生成    │
│ Web 阅读器              │      │ 跨书语义搜索             │
│ 基础搜索 + 标签          │      │ 知识图谱可视化           │
│ 多用户管理              │      │ 翻译增强 + 双语对照       │
│ 基础阅读统计             │      │ 高级阅读报告             │
│ Docker 一键部署          │      │ 优先技术支持             │
└────────────────────────┘      └────────────────────────┘
           免费                      $3-5/月 或 买断 $49
```

**切分合理性：**
- 免费层已经比 Calibre-Web 好用 → 吸引用户
- 付费层全是 **AI 计算密集型功能** → 有天然付费理由（用户理解 AI 要花钱）
- 用户可以自己接 API Key 免费用 AI 功能，付费 = 免配置 + 更好的体验

### 模式二：托管版 SaaS

```
自托管（免费）                     Omnigram Cloud（付费）
┌────────────────────────┐      ┌────────────────────────┐
│ 自己部署 Docker          │      │ 一键注册，无需服务器       │
│ 自己管理 + 更新          │      │ 自动备份 + CDN 加速       │
│ 全功能                  │      │ 全功能 + 移动端同步        │
│                        │      │ 5GB 免费 / 50GB $5/月    │
└────────────────────────┘      └────────────────────────┘
```

- **优势**：不是所有人都有 NAS / 会部署 Docker
- **风险**：存储成本高，电子书版权灰色地带

### 模式三：NAS 厂商合作 / OEM

```
与 NAS 厂商（群晖/威联通/极空间/绿联）谈预装合作
→ 成为官方「阅读」应用
→ 按设备授权收费 或 分成
```

- **优势**：一次合作 = 百万级用户曝光
- **风险**：谈判周期长，依赖厂商

### 模式四：企业/机构版

```
目标客户：小型图书馆、学校、企业内部培训
┌────────────────────────────┐
│ 多部门/多团队权限管理         │
│ LDAP/SSO 集成              │
│ 合规审计日志                 │
│ 批量导入 + 元数据管理         │
│ 定制 SLA 技术支持            │
│ 定价：$500-2000/年/实例      │
└────────────────────────────┘
```

## 推荐策略：分阶段推进

```
Phase 1（0-6个月）：纯开源，积累社区
├── 全功能免费开源（AGPL v3）
├── 目标：GitHub 1000+ stars
├── 进入 awesome-selfhosted / CasaOS 应用商店
└── 接受 GitHub Sponsors 捐赠

Phase 2（6-12个月）：引入 Pro 层
├── 核心功能保持免费
├── AI 增值功能 → Omnigram Pro
├── 定价 $3-5/月 或 买断 $49
└── 用户可选择自带 API Key 免费使用 AI

Phase 3（12-18个月）：多元化
├── 推出 Omnigram Cloud 托管版
├── 接洽 NAS 厂商合作
├── 探索企业/机构版
└── 目标：MRR $5000+
```

---

## 定价心理学

| 策略 | 原因 |
|------|------|
| AI 功能收费，阅读功能免费 | 用户理解「AI 要花钱」，不会觉得被割韭菜 |
| 提供「自带 API Key」选项 | 尊重开源用户，他们会成为你的布道者 |
| 年付打折（$49 买断 vs $5/月） | 降低决策门槛 |
| 个人免费，商用收费（AGPL） | AGPL 天然要求商业使用者付费或开源 |

## 许可证选择建议

```
MIT    → 太宽松，别人直接拿去卖
GPL v3 → 前端 Flutter 不太合适
AGPL   → ✅ 最佳选择
         ├── 个人自托管：完全免费
         ├── 商业使用：必须开源或购买商业许可
         └── 类似 Nextcloud / Outline 的策略
```

---

## 与此前战略的关系

- Open Core 模式与方向 A（AI-Native Calibre-Web）完美匹配
- AI 功能天然适合作为付费增值层（计算成本 = 付费理由）
- Phase 1 的开源社区积累是后续所有商业模式的基础
- AGPL 许可证既保护项目又允许个人自由使用
