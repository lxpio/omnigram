# 007 - 目标客户群体分析

> 讨论日期：2026-03-21
> 状态：✅ 已确认
> 核心问题：Omnigram 的目标客户是谁？如何分阶段触达？

## 用户画像坐标系

```
                    技术能力
                      ↑
                      │
         ② 技术极客    │    ① 数字囤积者
         (会折腾Docker) │    (NAS重度用户)
                      │
    ──────────────────┼──────────────────→ 阅读量
                      │
         ④ 轻度用户    │    ③ 深度阅读者
         (不是目标)    │    (知识工作者)
                      │
```

---

## 第一圈层：核心用户（早期采用者）

### ① NAS 数字囤积者

| 特征 | 描述 |
|------|------|
| **是谁** | 拥有群晖/威联通/DIY NAS，囤了几百GB电子书 |
| **痛点** | 书堆在文件夹里，找不到、管不好、只能下载到本地看 |
| **现在怎么办** | 用 Calibre-Web（难用）或不管（纯文件夹） |
| **为什么选 Omnigram** | 终于有个「像 Jellyfin 管视频一样管书」的工具 |
| **在哪里找他们** | r/selfhosted、r/DataHoarder、NAS 论坛、什么值得买 |
| **规模估计** | 全球 ~50-100万人 |

### ② 自托管技术极客

| 特征 | 描述 |
|------|------|
| **是谁** | 家里有 HomeLab，跑着 CasaOS/Unraid/Proxmox |
| **痛点** | Docker 生态里没有好的阅读管理应用 |
| **现在怎么办** | Calibre-Web + Audiobookshelf 分开部署 |
| **为什么选 Omnigram** | 一个容器搞定书库+阅读+TTS，AI 原生 |
| **在哪里找他们** | GitHub、Hacker News、r/homelab、v2ex |
| **规模估计** | 全球 ~30-50万人 |

---

## 第二圈层：扩展用户（产品成熟后）

### ③ 知识工作者 / 深度阅读者

| 特征 | 描述 |
|------|------|
| **是谁** | 程序员、研究者、作家、终身学习者，年读 20+ 本书 |
| **痛点** | 读了很多但记不住，笔记散落各处，无法跨书关联 |
| **现在怎么办** | Kindle + Readwise + Notion（月费 $20+） |
| **为什么选 Omnigram** | 自托管 = 数据自主，AI 跨书搜索，不用付月费 |
| **在哪里找他们** | Twitter/X 读书圈、少数派、即刻、ProductHunt |
| **规模估计** | 全球 ~500万人 |
| **注意** | 这群人技术能力参差不齐，需要 Cloud 托管版 |

### ④ 通勤听书族

| 特征 | 描述 |
|------|------|
| **是谁** | 每天通勤 1-2 小时，想听书但不想付 Audible 月费 |
| **痛点** | 有声书贵（$15/本），且很多书没有有声版 |
| **现在怎么办** | 喜马拉雅/Audible 订阅，或者不听 |
| **为什么选 Omnigram** | 任意电子书 → AI 生成高质量有声书，免费 |
| **在哪里找他们** | 播客社区、通勤族论坛 |
| **规模估计** | 全球 ~数千万人（但转化难） |
| **注意** | 这是 TTS 功能成熟后才能覆盖的群体 |

---

## 第三圈层：长尾用户（远期）

### ⑤ 小型图书馆 / 教育机构

| 特征 | 描述 |
|------|------|
| **是谁** | 社区图书馆、中小学、培训机构 |
| **痛点** | 商业图书管理系统贵，缺少 AI 辅助功能 |
| **为什么选 Omnigram** | 低成本 + AI 辅助阅读 + 多用户管理 |
| **定价** | 企业版 $500-2000/年 |

### ⑥ 外语学习者

| 特征 | 描述 |
|------|------|
| **是谁** | 想通过原版书学外语的人 |
| **痛点** | 原版书太难，查词打断阅读流 |
| **为什么选 Omnigram** | AI 翻译增强 + 双语对照 + TTS 正确发音 |

```
         获客难度低 ◄─────────────────► 获客难度高
              │                           │
  付费意愿高   │  ① NAS 囤积者             │  ⑤ 机构客户
              │  ② 技术极客               │
              │                           │
              │─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─│
              │                           │
  付费意愿低   │  ③ 知识工作者             │  ④ 通勤听书族
              │  ⑥ 外语学习者             │
              │                           │
```

**Phase 1 专注：① + ②**（获客容易 + 付费意愿高 + 是产品的天然传播者）

## 推广渠道匹配

| 阶段 | 目标用户 | 渠道 | 动作 |
|------|---------|------|------|
| Phase 1 | ① ② 技术用户 | GitHub / r/selfhosted / Hacker News | 开源 + awesome-selfhosted 收录 |
| Phase 1 | ① ② 技术用户 | CasaOS / Runtipi / Umbrel 应用商店 | 提交一键安装包 |
| Phase 2 | ③ 知识工作者 | ProductHunt / 少数派 / 即刻 | 产品发布 + 评测文章 |
| Phase 2 | ④ 听书族 | 社交媒体 / B站 | TTS 功能演示视频 |
| Phase 3 | ⑤ 机构客户 | 直销 / 教育展会 | 企业版推广 |

---

## 总结

> **先抓 NAS 极客和自托管玩家（他们是种子用户和传播者），再用 AI 阅读增强和 TTS 破圈到知识工作者和听书族。**
